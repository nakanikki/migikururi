# Pro Micro 経由キー送出について（実験機能・現状は目標未達）

> ⚠ この機能は**実験中で、期待した効果は出ていません**。当初「無遅延化」を
> 狙いましたが、実測の結果 SendInput（標準の送出方法）とほぼ同じ速度で、
> 物理キーボード級には届きませんでした（理由は末尾）。既定では無効なので
> 通常利用には影響しません。透明性のため、試したことと結果を残します。

「右くるり」には、CLIP STUDIO PAINT などでのキー送出遅延を減らせないか、
**市販のマイコンボード（Pro Micro）を物理キーボードとして使う**実験機能が
あります。特別な裏技やチートではなく、誰でも再現できる方法です。

---

## なぜ「無遅延」なのか（背景）

Windows のソフトから他アプリへキーを送る標準手段が `SendInput` API です。
右くるりも通常はこれを使います。ところが CLIP STUDIO PAINT では、
**SendInput で送られたキー（injected input）だけ反映が 5〜6 フレーム遅れる**
という現象があります。

これはアプリ／OS 側の「注入された入力」の処理経路によるもので、送り方
（VK 方式・スキャンコード方式など）を工夫しても大きくは縮みません。
実測でも次のとおりでした（60fps 動画のフレーム差で計測）:

| 送出方法 | クリスタ反映 |
|---|---|
| 物理キーボード（ロジクール等） | 1〜2F |
| SendInput（VK 方式） | 6F |
| SendInput（スキャンコード方式） | 5F |
| **Pro Micro（本物の USB キーボードとして送出）** | **2F** |

…という仮説で Pro Micro を試しました。**しかし結果は期待外れ**でした（下記）。

## 計測結果（同条件・キー監視ツール表示を起点にフレーム数を計測）

| 送出方法 | クリスタ反映 |
|---|---|
| 物理キーボード（ロジクール等） | 1〜2F |
| SendInput（標準の送出） | 5F |
| **Pro Micro 直打ち（チップ最速）** | **3F** |
| **Pro Micro 経由（このソフトから）** | **5〜6F** |

**Pro Micro（ATmega32U4）はチップ自体の反映が直打ちでも 3F** あり、物理
キーボード（1F）級には届きませんでした。さらにソフトから USB シリアルで
指示を送る経路のオーバーヘッドで 5〜6F になり、**SendInput（5F）とほぼ
変わらない**という結論です（当初 2F と誤認しましたが、計測の起点を取り
違えていました）。

## 仕組み（何をしているか）

1. 市販の **Pro Micro（ATmega32U4）** という小さなマイコンボードを USB で挿す。
   これは Arduino Leonardo 互換で、**それ自体が USB キーボードとして
   振る舞える**（工作・自作キーボードで一般的に使われるもの）。
2. Pro Micro に「PC からの指示で対応するキーを打つ」だけのプログラムを書き込む。
3. 右くるりは、キーを送るとき `SendInput` の代わりに、**この Pro Micro へ
   シリアル通信で「このキーを打って」と伝える**。Pro Micro が物理的に
   キーを打つ。

結果として、クリスタから見れば**普通の外付けキーボードが打った**のと
まったく同じ入力になります。だから物理キーボードと同じ速さ（2F）で反映されます。

**隠していること・偽装していることは一切ありません。** むしろ逆で、
「ソフトからの注入」をやめて「本物のキーボード入力」にしているだけです。

---

## この機能を使うのに必要なもの

- **Pro Micro（ATmega32U4）** … 数百円〜のマイコンボード（Amazon 等で入手可）
- ボードにプログラムを書き込む手間（Arduino IDE。手順は下記）

**ハードが必要なので、右くるりの配布版では標準では無効**です。
config を編集して有効化した人だけが使えます（＝作者や、同じ工作をした人向け）。
一般ユーザーは何もしなくても従来どおり `SendInput` で動作します。

---

## 有効化の手順（再現方法）

### 1. Pro Micro にファームを書き込む

書き込むプログラムは公開しています → [tools/promicro/promicro_probe/promicro_probe.ino](../tools/promicro/promicro_probe/promicro_probe.ino)

Arduino IDE で開き、ボード「Arduino Leonardo」、対象の COM ポートを選んで
書き込むだけです（`Keyboard` ライブラリが必要）。

やっていることはソースを読めば分かるとおり、
**「シリアルで受け取ったキー指示を Keyboard.press／release で打つ」だけ**です。

### 2. どの COM ポートか調べる

デバイスマネージャー、または Arduino IDE のポート一覧で、Pro Micro
（Arduino Leonardo）の COM 番号を確認します（例: COM11）。

### 3. config を編集して有効化

右くるりの `config.json`（設定フォルダ内。トレイ →「設定フォルダを開く」
などで開ける）をテキストエディタで開き、次の1行を足します:

```json
"hid_port": "COM11"
```

（COM11 は手順2で調べた番号に置き換え）

右くるりを再起動すると、以降キー送出が Pro Micro 経由になります。
`hid_port` を空 `""` にするか削除すれば、従来の `SendInput` に戻ります。

## まとめ・現状

- 仕組みは **Pro Micro を本物の USB キーボードとして使っているだけ**。ソースも
  ファームも手順も全公開で、偽装・隠蔽は一切なし（むしろ「ソフトの注入を
  やめて物理入力にする」透明な方法）。
- **ただし効果は目標未達**。ATmega32U4 チップ自体の反映が直打ちでも 3F あり、
  ソフト経由だと 5〜6F で SendInput（5F）とほぼ同じ。物理キーボード（1F）
  級にはならなかった。
- 使用した実機: **FORIOT の Pro Micro（Type-C・ATmega32U4・5V/16MHz）**。
- 1F 級を狙うなら別チップ（nRF52840 系など、実測で 1F だったもの）が必要。
  この機能は実験として残すが、現状フレーム短縮の主張はしない。
